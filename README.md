# sabu-watch — 사회부 당번용 속보 감시기

네이버 뉴스 검색 API + 구글 뉴스 RSS를 3분마다 폴링해서
사회부 관련 **[단독] / [속보] / 주요 사건사고** 기사가 뜨면
macOS 배너 알림(사운드 포함)을 즉시 보낸다.

## 실행 방식 — GitHub Actions 클라우드 (기본)

맥북이 꺼져 있어도 **GitHub Actions가 5분 간격으로 감시기를 실행**해 텔레그램으로
알림을 보낸다 (저장소: `seungyeoncho95-droid/sabu-watch`, 공개).
자격증명은 저장소 Actions Secrets에 등록돼 있고, 중복 방지 기록(`state.json`)은
실행마다 저장소에 커밋해서 이어간다.

```bash
gh workflow disable sabu-watch   # 감시 끄기 (당번 아닌 날 등)
gh workflow enable sabu-watch    # 감시 켜기
gh run list --limit 5            # 최근 실행 확인
```

주의: 깃허브 스케줄 특성상 알림이 5~15분가량 지연될 수 있다.
공개 저장소라 코드·키워드는 노출되지만 자격증명(Secrets)은 비공개.

## 로컬 실행 (보조 — 클라우드와 동시 실행 시 중복 알림 발생)

```bash
cd ~/Desktop/sabu-watch
python3 watch.py                 # 감시 시작 (Ctrl+C로 종료, 3분 간격이라 더 빠름)
python3 watch.py --once          # 1회만 실행 (테스트)
python3 watch.py --interval 120  # 폴링 간격 변경(초)
```

로컬로 돌릴 땐 먼저 `gh workflow disable sabu-watch`로 클라우드를 끌 것.
로컬과 클라우드의 state.json이 따로 놀아서 같은 기사가 두 번 알림된다.

## 작동 방식

1. **수집** — 네이버 뉴스 API(`단독`, `속보`, `화재`, `붕괴`, `추락 사망`, `흉기`, `실종`, `폭발 사고`)를
   매 사이클 조회. **네이버 뉴스에 송고된 기사만** 대상(알림 링크가 n.news.naver.com).
   네이버 미송고 기사는 알림하지 않음. 구글 RSS는 네이버 링크를 줄 수 없어 제거(2026-07-12).
2. **판별** — 아래 셋 중 하나면 알림 대상:
   - 제목에 `[단독]` 머리표 + 사회부 키워드(법조/경찰·사건/재난/행정) 매칭
   - 제목에 `[속보]`/`[긴급]` 머리표 + 사회부 키워드 매칭
   - 머리표 없어도 사망·실종·붕괴·폭발 등 심각성 어휘가 제목에 있으면 (칼럼·인터뷰·의문형 제목 제외)
   - 증시·연예·스포츠 등 노이즈 키워드가 제목에 있으면 제외
3. **중복 제거(사건 클러스터)** — 링크·정규화 제목으로 1차 걸러낸 뒤,
   알림 나간 사건마다 클러스터를 만들어 병합된 기사들의 어휘(제목 2글자 조각)를 계속 흡수.
   새 기사 어휘의 38% 이상이 클러스터에 포함되면 같은 사건의 후속·타 매체 기사로
   보고 병합(첫 기사만 알림, 나머지는 `alerts.log`에만 기록). "승조원→일병→병사",
   "고성→동해→NLL"처럼 표현이 바뀌는 후속 기사도 잡힌다. 클러스터는 마지막 병합
   시점 기준 72시간 유지되므로 진행 중인 사건은 계속 억제된다.
4. **알림** — 텔레그램(@dangbon_bot). 형식: `@언론사명/제목=소관기관` + 네이버 기사 링크.
   언론사명은 네이버 기사 페이지 og:article:author 메타에서, 소관기관은 제목·요약의
   ○○경찰서/○○지법/○○지검/○○구청 등 패턴에서 추출(없으면 대분류로 대체).
   텔레그램 미설정·전송 실패 시
   macOS 배너 + Glass 사운드로 폴백. 사이클당 최대 8건(초과분은 터미널에만).
   텔레그램 설정: `.env`에 `TELEGRAM_BOT_TOKEN` 넣고 봇에게 메시지 하나 보낸 뒤
   `python3 watch.py --setup-telegram` (chat_id 자동 등록 + 테스트 발송).

## 파일

- `watch.py` — 전체 로직 (단일 파일, 표준 라이브러리만 사용)
- `state.json` — 이미 본 기사 기록 (72시간 보관). 지우면 처음부터 다시 감시
- `alerts.log` — 알림 이력 + 병합된 유사기사 링크
- 네이버 자격증명은 `~/community-scraper/.env`의 `NAVER_CLIENT_ID/SECRET`을 재사용
  (이 폴더에 `.env`를 만들면 그쪽이 우선)

## 튜닝 포인트 (watch.py 상단 상수)

- `SOCIAL_KEYWORDS` / `EXCLUDE_KEYWORDS` — 관심/제외 키워드
- `NAVER_QUERIES` — 검색 질의 추가 (예: 특정 지역명, 담당 라인)
- `MAX_ALERTS_PER_CYCLE`, `LOOKBACK_MIN` — 알림 상한·허용 기사 나이
- `CLUSTER_THRESHOLD = 0.38` — 낮추면 병합 공격적(놓치는 후속기사 감소, 별개 사건
  오병합 위험 증가), 높이면 보수적. 실측상 같은 사건 후속기사는 0.43~0.94,
  별개 사건은 0.3 미만이라 그 사이 값
